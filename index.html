Domain Authentication Setup

Why @anmng.com emails aren't being delivered

When emails work for Gmail but not for your custom domain (@anmng.com), it's typically because:

  • Your domain lacks proper SPF/DKIM records
  • Formspree isn't authorized to send emails for your domain
  • Your email server has strict spam filters
  • DNS changes haven't fully propagated

Step 1: Log in to Formspree

1. Go to the Formspree Dashboard

2. Navigate to the "Domains" section

3. Click "Add Domain" and enter "anmng.com"

Step 2: Verify Domain Ownership

Formspree will provide a TXT record to verify you own the domain:

Type: TXT
Name: @ or anmng.com
Value: formspree-verification=xxxxxxxxxxxxxxxxxxxx

Add this record to your DNS configuration (where you manage your domain).

Step 3: Set Up Email Authentication

To ensure deliverability to @anmng.com, add these records:

Type Name/Host Value Priority
SPF (TXT) @ v=spf1 include:spf.formspree.io ~all -
DKIM (CNAME) formspree._domainkey formspree.dkim.freshdesk.com -
DMARC (TXT) _dmarc v=DMARC1; p=none; rua=mailto:admin@anmng.com; -

DNS Configuration Guide

Where to Add DNS Records

DNS records are managed through your domain registrar or hosting provider:

GoDaddy: Domain Manager → DNS Management
Namecheap: Domain List → Manage → Advanced DNS
Cloudflare: DNS → Records

SPF Record Configuration

SPF (Sender Policy Framework) tells email servers which senders are authorized for your domain:

# SPF Record for anmng.com
Type: TXT
Host: @
Value: v=spf1 include:spf.formspree.io ~all

DKIM Record Configuration

DKIM (DomainKeys Identified Mail) adds a digital signature to verify email authenticity:

# DKIM Record for anmng.com
Type: CNAME
Host: formspree._domainkey
Value: formspree.dkim.freshdesk.com

DMARC Record Configuration

DMARC (Domain-based Message Authentication) tells receiving servers what to do with unauthenticated emails:

# DMARC Record for anmng.com
Type: TXT
Host: _dmarc
Value: v=DMARC1; p=none; rua=mailto:admin@anmng.com;

Troubleshooting Domain Delivery

Common Issues and Solutions

1

Emails not received in @anmng.com inbox

Possible solutions:

  • Check spam folder: Mark Formspree as "Not Spam"
  • Whitelist Formspree: Add noreply@formspree.io to your safe senders list
  • Verify DNS: Ensure SPF/DKIM records are correct
2

DNS configuration errors

How to diagnose:

  • Use MXToolbox to verify SPF/DKIM records
  • Check for typos in DNS values
  • Allow 24-48 hours for DNS propagation
3

Email server configuration issues

For self-hosted email servers:

  • Check server firewall settings
  • Review SMTP relay configuration
  • Examine mail server logs for rejection reasons
4

Formspree domain authentication

Verify in Formspree dashboard:

  • Domain shows as "Verified"
  • SPF and DKIM show green checkmarks
  • No warnings or errors in domain status
