MX Record Configuration Guide

Step-by-step instructions to configure your MX records for sankalppankaj@anmng.com

Step 1: Access DNS Management

Log in to your domain registrar or DNS hosting provider:

GoDaddy
Namecheap
Cloudflare
Google Domains

Navigate to the DNS management section for your domain (anmng.com).

Step 2: Add MX Records

Create these MX records in your DNS settings:

Type Name/Host Value/Points to Priority TTL
MX @ mail.anmng.com 10 3600
MX @ backup.anmng.com 20 3600
Priority 10 is your primary mail server

Step 3: Save Changes

After adding the records, save your changes. DNS updates can take 24-48 hours to propagate globally.

Note: Some providers require you to remove existing MX records before adding new ones.

Step 4: Verify Configuration

Use the tester tool on this page to verify your MX records are correctly configured.

You can also use external tools like MXToolbox to check your records.

How MX Records Work

MX records direct email traffic to your mail servers:

Email Sent

Someone sends an email to sankalppankaj@anmng.com

DNS Query

The sending server looks up MX records for anmng.com

Server Selection

The server with the lowest priority (10) is tried first

Email Delivery

Email is delivered to your mail server

Backup Server

If primary fails, the backup server (priority 20) is used

Common Issues

  • Incorrect priority values (lower number = higher priority)
  • Missing the @ symbol for the root domain
  • Forgetting to remove old MX records
  • TTL set too high, delaying propagation
  • Typos in the mail server address
